原码、反码、补码及字符编码

深入讲解计算机中数据的表示方法，包括原码、反码、补码的转换规则以及常见的字符编码方式，为CSP初赛做好准备。

学习概念 查看例题

机器数与编码方式

机器数与真值

计算机中用二进制表示的数称为机器数，它包含符号位和数值位两部分：

  • 符号位：通常用最高位表示，0表示正数，1表示负数
  • 数值位：表示数的大小
  • 真值：带符号的机器数所表示的实际数值

示例：8位机器数10000010，最高位1表示负数，其真值为-2

1. 原码

原码是最简单的机器数表示法，其规则为：

  • 符号位：正数为0，负数为1
  • 数值位：该数的绝对值的二进制表示

原码表示示例（8位）：

+5的原码：

0 0 0 0 0 1 0 1

-5的原码：

1 0 0 0 0 1 0 1

特点：原码表示直观，但进行加减法运算时需要额外处理符号位，增加了硬件复杂度。

2. 反码

反码是原码向补码过渡的编码方式，其规则为：

  • 正数的反码与原码相同
  • 负数的反码：符号位不变，数值位按位取反（0变1，1变0）

反码表示示例（8位）：

+5的反码：

0 0 0 0 0 1 0 1

与原码相同

-5的反码：

1 1 1 1 1 0 1 0

符号位不变，数值位取反

3. 补码

补码是计算机中最常用的编码方式，解决了原码和反码的运算问题，其规则为：

  • 正数的补码与原码相同
  • 负数的补码：其反码加1（末位加1，有进位则进位）

补码表示示例（8位）：

+5的补码：

0 0 0 0 0 1 0 1

与原码、反码相同

-5的补码：

1 1 1 1 1 0 1 1

反码加1得到

补码优势：可以将减法运算转化为加法运算，符号位可以直接参与运算，简化了计算机硬件设计。

4. 字符编码

字符编码用于将字符转换为计算机可以识别的二进制数，CSP初赛常考的编码方式有：

ASCII编码

  • 美国信息交换标准代码
  • 使用7位二进制表示，共128个字符
  • 包括大小写字母、数字、标点符号和控制字符
  • 数字'0'的ASCII码是48（0x30）
  • 大写字母'A'的ASCII码是65（0x41）
  • 小写字母'a'的ASCII码是97（0x61）

Unicode编码

  • 统一码，为世界上所有字符制定了唯一编码
  • UTF-8是Unicode的一种实现方式，可变长度
  • 汉字通常用3个字节表示
  • 兼容ASCII编码（0-127范围相同）

CSP初赛解题技巧

原码、反码、补码计算技巧

  • 快速转换法：负数补码 = 绝对值原码按位取反 + 1
  • 补码转原码：对补码再求一次补码即可得到原码
  • 8位整数范围：
    • 原码：-127 ~ +127
    • 反码：-127 ~ +127
    • 补码：-128 ~ +127（注意-128的特殊性）
  • 简便计算：计算补码时，从右向左找到第一个1，该位不变，左边所有位取反，右边（包括该位）不变

ASCII编码解题要点

  • 大小写转换：大写字母ASCII码值 + 32 = 对应小写字母
  • 字符比较：在程序中，字符比较实际是比较它们的ASCII码值
  • 数字字符转换：数字字符'0'-'9'减去48（或'0'）可得对应整数
  • 常见考点：记住几个关键值（'0'=48, 'A'=65, 'a'=97）

解题步骤

  1. 明确题目考查的是哪种编码方式
  2. 确定数值的位数（通常是8位，CSP初赛常见）
  3. 对于负数，严格按照转换规则逐步计算
  4. 检查结果是否符合该编码方式的数值范围
  5. 字符编码题先确定使用的编码标准（ASCII或Unicode）

CSP初赛例题解析

例题1：补码计算

题目：在8位二进制补码中，-128的补码表示为？

A. 10000000

B. 11111111

C. 00000000

D. 11111110

解析

在8位二进制补码中：

  • 范围是-128到+127
  • -128是一个特殊值，没有对应的原码和反码
  • 其补码规定为10000000

答案：A

例题2：ASCII码计算

题目：已知字符'A'的ASCII码值为65，则字符'C'的ASCII码值的十六进制表示为？

A. 0x41

B. 0x42

C. 0x43

D. 0x44

解析

字符在ASCII表中按顺序排列：

  • 'A'的ASCII码是65（0x41）
  • 'B'的ASCII码是66（0x42）
  • 'C'的ASCII码是67（0x43）
  • 十六进制43对应十进制67

答案：C

例题3：编码转换

题目：8位二进制数10010101作为补码时，对应的十进制数是多少？

A. -107

B. -109

C. 149

D. -149

解析

补码转十进制步骤：

  1. 最高位为1，说明是负数
  2. 对补码求补码得到原码：11101011
  3. 数值位1101011对应的十进制是107
  4. 所以该补码表示的十进制数是-107

答案：A

例题4：编码应用

题目：以下程序段的输出结果是？

#include <iostream>
using namespace std;

int main() {
    char c1 = 'A', c2 = 'a';
    cout << (int)c1 << " " << (int)c2 << " " << (c2 - c1);
    return 0;
}

A. 65 97 32

B. 65 97 -32

C. A a 32

D. 65 97 0

解析

程序分析：

  • 'A'的ASCII码是65，'a'的ASCII码是97
  • (int)c1将字符转换为对应的ASCII码值
  • c2 - c1 = 97 - 65 = 32
  • 所以输出结果是65 97 32

答案：A
